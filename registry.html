NANDA Registry

The NANDA Registry provides a robust infrastructure for discovering, authenticating, and verifying agents, servers, and resources in the decentralized AI network. It extends the capabilities of MCP by enabling large-scale agent ecosystems to connect and interact with confidence through standard discovery mechanisms.

The NANDA Registry architecture showing the relationship between discovery, authentication, verification, and analytics services

Key Features

The NANDA Registry provides critical infrastructure that connects agents, resources, and tools across the decentralized network:

Semantic Discovery

Advanced search capabilities that enable agents to find relevant services based on semantic understanding of capabilities, not just keyword matching. The registry supports natural language queries to identify agents with specific abilities.

Verification System

Comprehensive ownership verification process that establishes trust in agent identities and capabilities through DNS verification, file upload validation, and meta tag confirmation.

Reputation Tracking

Transparent reputation metrics based on uptime, response times, error rates, and user ratings that help agents identify reliable partners in the network.

Analytics Services

Detailed usage analytics and performance metrics across the entire network, allowing operators to monitor and optimize their agents.

Webhook Notifications

Event-based notification system that allows services to react in real-time to changes in the network such as new agent registrations, capability updates, or status changes.

Recommendation Engine

Personalized agent recommendations based on usage history, capabilities, and compatibility metrics that facilitate efficient service discovery.

Registry Architecture

The NANDA Registry's architecture consists of several interconnected services that manage different aspects of the agent ecosystem:

Server Registry Service

Core database of all registered agents, resources, and tools with comprehensive metadata about their capabilities, protocols, and requirements.

Authentication Service

Handles identity verification, API key management, and access control for interactions with registry services.

Verification Service

Conducts ownership verification, capability checks, and security assessments to establish the authenticity and trustworthiness of registered agents.

Discovery Service

Semantic search and recommendation capabilities that help agents find matching services based on functionality, performance, or compatibility needs.

Analytics Service

Collects and analyzes usage data across the network to provide insights about performance, popularity, and trends.

Webhook Service

Event notification system that allows asynchronous communication about registry events to subscribed endpoints.

API Endpoints

The NANDA Registry provides a RESTful API with the following core endpoints:

Server Registration and Management

POST/api/v1/servers/

Register a new NANDA-compatible agent, resource, or tool in the registry with detailed metadata about capabilities, protocols, and usage requirements.

GET/api/v1/servers/{id}/

Retrieve detailed information about a specific registered server, including its capabilities, verification status, and performance metrics.

PUT/api/v1/servers/{id}/

Update the details of an existing server to reflect changes in capabilities, endpoints, or other metadata.

Discovery and Search

GET/api/v1/discovery/search/

Advanced semantic search for discovering agents based on capabilities, functionality, tags, verification status, and other criteria.

GET/api/v1/discovery/recommend/

Get personalized recommendations for agents based on usage history, preferences, and popularity metrics.

GET/api/v1/discovery/popular/

List the most popular agents in the network based on usage statistics and ratings over different time periods.

Verification

POST/api/v1/verification/request/{server_id}/

Initiate the verification process for an agent to establish ownership and validate claimed capabilities.

POST/api/v1/verification/complete/{verification_id}/

Complete the verification process by providing necessary proof through DNS records, file uploads, or meta tags.

GET/api/v1/verification/status/{verification_id}/

Check the status of a server verification request and view the results of individual verification checks.

Analytics

GET/api/v1/analytics/network/

Retrieve analytics about the overall NANDA network, including total servers, request volumes, and active agents.

GET/api/v1/analytics/servers/{server_id}/

Get detailed usage analytics for a specific server, including request counts, response times, and error rates.

POST/api/v1/analytics/log/

Record request details for analytics tracking and performance monitoring within the network.

Webhooks

POST/api/v1/webhooks/

Create a new webhook for receiving notifications about registry events like server creation, updates, or verification status changes.

GET/api/v1/webhooks/{id}/

Get details about a specific webhook configuration, including subscribed events and delivery history.

POST/api/v1/webhooks/{webhook_id}/test/

Send a test event to a webhook to verify it's working correctly before relying on it for important notifications.

Agent Discovery

The NANDA Registry enables sophisticated agent discovery through several mechanisms:

Semantic Search

The registry provides advanced semantic search capabilities that understand the intent behind search queries. Agents can find relevant services by describing what they need in natural language, not just through exact keyword matches.


// Example search query
const searchResults = await nandaClient.discovery.search({
  q: "Image generation agents that support realistic landscape creation with style transfer",
  verified: true,
  limit: 10
});

Capability-Based Matching

Agents can query the registry for specific capabilities they need, and the registry will match them with servers that provide those exact functionalities.


// Example capability-based query
const imageAgents = await nandaClient.discovery.search({
  q: "image-generation",
  tags: "landscape,photorealistic,high-resolution",
  types: "agent"
});

Personalized Recommendations

Based on an agent's usage history and preferences, the registry can recommend compatible and high-performing agents tailored to its specific needs.


// Example recommendation query
const recommendations = await nandaClient.discovery.recommend({
  limit: 5,
  type: "resource"
});

Verification Mechanism

Trust is essential in a decentralized agent network. The NANDA Registry includes a robust verification process to establish ownership and validate capabilities:

  1. Verification Request

    An agent owner initiates verification by submitting a formal request through the registry API.

  2. Ownership Verification

    The registry generates a unique verification token that must be proven through one of several methods:

    • DNS Verification: Adding a specific TXT record to the domain's DNS settings
    • File Verification: Uploading a file with the verification token at a specific URL
    • Meta Tag Verification: Adding a meta tag containing the token to the homepage HTML

  3. Capability Verification

    The registry tests the agent's claimed capabilities through automated checks and example invocations.

  4. Security Assessment

    Basic security evaluations ensure the agent follows best practices for authentication and data protection.

  5. Health Monitoring

    Ongoing health checks verify uptime, response times, and reliability after verification.

Verified Badge

Successfully verified agents receive a digital badge and special status in the registry, increasing visibility and trust within the network.

Analytics & Reporting

The NANDA Registry provides comprehensive analytics to help understand network activity and agent performance:

Network-Level Analytics

Aggregate metrics about the entire NANDA ecosystem, including:

  • Total registered agents by type (agent, resource, tool)
  • Overall request volumes and trends
  • Distribution of agent types and capabilities
  • Growth rates and adoption metrics

Server-Specific Analytics

Detailed performance metrics for individual agents:

  • Request volumes and patterns over time
  • Average response times and distribution
  • Error rates and common error types
  • Client diversity and usage patterns
  • Uptime and reliability statistics

Usage Reporting

Structured logs of agent interactions to enable billing, debugging, and optimization:

  • Detailed request and response information
  • Capability utilization statistics
  • Geographic distribution of requests
  • Client identification and tracking

Webhooks & Notifications

To enable real-time reactivity within the network, the NANDA Registry provides a flexible webhook system:

Event Types

The registry can notify subscribers about various events:

  • server.created: When a new agent is registered
  • server.updated: When an agent's metadata is modified
  • server.deleted: When an agent is removed from the registry
  • server.verified: When an agent completes verification
  • verification.requested: When verification is initiated
  • verification.completed: When verification process concludes
  • server.status_changed: When an agent's status changes

Security Features

Webhooks include security measures to ensure notification integrity:

  • Unique secrets for signing payload contents
  • Signature verification to confirm authenticity
  • Automatic retries for failed deliveries
  • Delivery logs for auditing and debugging

// Example webhook registration
const webhook = await nandaClient.webhooks.create({
  url: "https://my-agent-server.example/webhooks/registry",
  events: ["server.verified", "server.status_changed"],
  description: "Monitor verification status changes"
});

Hosting a NANDA Registry

One of the core strengths of NANDA is its decentralized architecture. Any institution, company, or individual can host their own NANDA Registry instance, contributing to the resilience and diversity of the agent ecosystem.

Why Host Your Own Registry?

  • Data Sovereignty: Maintain control over your agent registry data within your own infrastructure
  • Customization: Tailor the registry to your specific industry needs or organizational requirements
  • Private Networks: Create closed agent ecosystems for enterprise or internal use cases
  • Geographic Distribution: Improve performance by hosting registry instances closer to your users
  • Resilience: Contribute to the decentralized nature of NANDA by adding an additional registry node

Deployment Options

Docker Deployment (Recommended)

The fastest way to get a registry up and running with minimal configuration.

Prerequisites:

  • Docker and Docker Compose
  • Domain name (for production deployment)
  • SSL certificate (Let's Encrypt recommended)

Basic Setup:

  1. Clone the repository:
    git clone https://github.com/aidecentralized/nanda-registry.git
    cd nanda-registry
  2. Configure environment variables in .env file
  3. Customize Nginx configuration
  4. Build and run containers:
    docker-compose up -d
  5. Create admin user:
    docker-compose exec web python manage.py createsuperuser
  6. Set up SSL with Let's Encrypt

AWS Elastic Beanstalk

Ideal for organizations already using AWS infrastructure with auto-scaling capabilities.

Prerequisites:

  • AWS Account
  • AWS CLI configured
  • EB CLI installed

Basic Setup:

  1. Clone the repository
  2. Create an Elastic Beanstalk environment:
    eb init -p docker
    eb create nanda-registry-env
  3. Set environment variables
  4. Deploy the application:
    eb deploy
  5. Configure domain with Route 53

Registry Configuration

The NANDA Registry is highly configurable through environment variables and custom settings:

Key Environment Variables

# Django Core
SECRET_KEY=your-secret-key
DEBUG=False
ALLOWED_HOSTS=your-domain.com

# Database Settings
DB_NAME=nanda_db
DB_USER=postgres
DB_PASSWORD=[strong-password]
DB_HOST=db
DB_PORT=5432

# Redis & Celery
REDIS_URL=redis://redis:6379/0
CELERY_BROKER_URL=redis://redis:6379/1
CELERY_RESULT_BACKEND=redis://redis:6379/2

# Email Configuration
EMAIL_HOST=smtp.example.com
EMAIL_PORT=587
EMAIL_HOST_USER=your-email@example.com
EMAIL_HOST_PASSWORD=your-email-password
EMAIL_USE_TLS=True
DEFAULT_FROM_EMAIL=noreply@your-domain.com

# Frontend URL
FRONTEND_URL=https://your-domain.com

Advanced Settings

Additional settings can be configured by editing mcp_nexus/settings.py:

  • VERIFICATION_TOKEN_EXPIRY: Duration for verification tokens
  • VERIFICATION_CHECK_INTERVAL: Interval for server health checks
  • ANALYTICS_RETENTION_DAYS: Days to retain analytics data

Maintaining Your Registry

Database Backups

Set up regular database backups to prevent data loss:

For Docker deployment:

docker-compose exec db pg_dump -U postgres mcp_nexus > backup_$(date +%Y%m%d).sql

For AWS deployment:

aws rds create-db-snapshot --db-instance-identifier your-db-identifier --db-snapshot-identifier snapshot-name

Updates

Keep your registry up to date with the latest features and security patches:

For Docker deployment:

git pull
docker-compose down
docker-compose build
docker-compose up -d

For AWS deployment:

git pull
eb deploy

Security Best Practices

  • Regularly update dependencies
  • Set strong passwords for database and admin users
  • Keep DEBUG=False in production
  • Configure proper HTTPS
  • Implement rate limiting for public APIs
  • Consider adding WAF protection

When your registry is running, comprehensive API documentation is available at:

  • Swagger UI: /api/v1/schema/swagger-ui/
  • ReDoc: /api/v1/schema/redoc/

Implementation Guidelines

When implementing NANDA Registry integration, consider the following guidelines:

Discovery First

Design your agents to dynamically discover capabilities rather than hardcoding connections. This creates more resilient and adaptable AI systems that can find the best available services as the network evolves.

Detailed Registration

Provide comprehensive metadata during registration, including clear capability descriptions, accurate parameter specifications, and example usage. This improves discovery and matching for your agent.

Verification Priority

Complete the verification process early to establish trust in the network. Verified agents receive preferential ranking in search results and recommendations.

Analytics Integration

Implement proper analytics logging to contribute to and benefit from the network's intelligence about performance and usage patterns.
