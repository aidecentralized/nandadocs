NANDA SDK

The NANDA SDK provides developers with the tools needed to build, register, and connect AI agents within the NANDA ecosystem. It simplifies integration with the registry, enables server verification, and facilitates agent-to-agent communication.

Development Status: Coming Soon

The NANDA SDK is currently under active development and will be released soon. Early-access preview versions are available to contributors. Core functionality for node creation, registry integration, identity verification, and reputation checking are being implemented.

Key Features

The NANDA SDK is being designed to provide these essential capabilities for NANDA-compatible agents:

Node Creation

Easily create and configure NANDA-compatible nodes that can function as both clients and servers in the network.

Registry Integration

Register nodes with the NANDA Registry to make them discoverable to other agents in the network.

Identity Verification

Verify the identity of other nodes against the registry to ensure secure and trusted interactions.

Reputation Checking

Query and evaluate the reputation of agents in the network to make informed decisions about collaborations.

Protocol Handling

Simplified interfaces for handling NANDA Protocol messages, removing the complexity of raw JSON-RPC.

Security Utilities

Built-in tools for generating and validating certificates, keys, and other security credentials.

Quickstart Preview

While the SDK is not yet publicly released, here's a preview of how you'll be able to use it once available:

Creating and Registering a NANDA Node

import { NandaNode, Registry } from '@nanda/sdk'; // Create a new NANDA node const myNode = new NandaNode({ name: 'MyAIAgent', description: 'A specialized agent for data analysis', capabilities: ['data-analysis', 'visualization', 'forecasting'], endpoint: 'https://my-agent-api.example.com' }); // Generate identity credentials await myNode.generateIdentity(); // Connect to the NANDA Registry const registry = new Registry({ url: 'https://registry.nanda-protocol.org' }); // Register the node with the registry const registrationResult = await registry.registerNode(myNode); console.log(`Node registered with ID: ${registrationResult.nodeId}`);

Discovering and Verifying Other Nodes

// Discover nodes with specific capabilities const translationAgents = await registry.discoverNodes({ capabilities: ['translation'], minReputationScore: 4.0 }); // Connect to the first result const targetNode = translationAgents[0]; const connection = await myNode.connect(targetNode.endpoint); // Verify the node's identity against the registry const isVerified = await connection.verifyIdentity(); if (!isVerified) { console.warn('Could not verify node identity!'); connection.disconnect(); } // Check the node's reputation const reputationData = await registry.getReputation(targetNode.id); console.log(`Node reputation: ${reputationData.score}/5.0 (${reputationData.reviewCount} reviews)`);

Node Creation

When released, the SDK will provide a straightforward way to create NANDA-compatible nodes that can participate in the network. These nodes can function as clients, servers, or both, with the ability to:

  • Define their capabilities and metadata
  • Implement custom functionality through handler registration
  • Manage multiple connections with other nodes
  • Handle protocol messages automatically

Registry Integration

The SDK will simplify the process of registering nodes with the NANDA Registry, making them discoverable to other agents in the network. This will include:

  • Automated registration flows
  • Metadata management and updates
  • Capability advertisement
  • Endpoint configuration

Identity Verification

A key feature of the upcoming SDK will be robust identity verification capabilities. This will allow nodes to:

  • Generate cryptographic identity credentials
  • Verify the identity of other nodes against the registry
  • Establish trusted connections
  • Maintain a web of trust within the network

Reputation System

The SDK will integrate with NANDA's reputation system, enabling nodes to:

  • Query reputation data for other nodes
  • Report performance metrics and feedback
  • Make trust-based decisions about collaborations
  • Build their own reputation through reliable service
