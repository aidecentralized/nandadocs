NANDA SDK

The NANDA SDK provides developers with the tools needed to build, register, and connect AI agents within the NANDA ecosystem. It simplifies integration with the registry, enables server verification, and facilitates agent-to-agent communication.

The NANDA SDK architecture showing connections between agents, registry, and verification services

Key Features

The NANDA SDK is designed to simplify the development and integration of agents in the NANDA ecosystem:

Server Registration

Tools for registering servers with the NANDA Registry, making them discoverable to other agents in the network.

Identity Management

Certificate generation and validation capabilities for establishing secure, verifiable agent identities.

Verification Integration

Verification tools that connect with the NANDA Registry to validate server ownership and capabilities.

Discovery Client

Mechanisms that allow agents to locate and connect with other agents based on capability requirements.

Feedback Integration

Tools for submitting and consuming reputation data to build trust in the network.

Analytics Reporting

Analytics capabilities for tracking agent usage, performance metrics, and interaction patterns.

Installation

The NANDA SDK will be available as an npm package for JavaScript/TypeScript applications:

npm
yarn
npm install @nanda/sdk

Version Roadmap

The NANDA SDK is being developed in phases, with each version adding new capabilities:

Version 1: Basic Registration

Current Release

Objective: Support for registering servers

Enable basic server registration with the NANDA Registry, allowing agents to be discoverable in the network.

Key Features:

  • API client for Registry interaction
  • Server registration functionality
  • Basic server discovery

Demo Objective:

Allow anyone to add their server to the host

Coming Soon

Key Features:

Demo Objective:

In Development

Server Registration

The SDK will provide simple interfaces for registering servers with the NANDA Registry, allowing them to be discovered by other agents in the network. This will include functions for creating, updating, and managing server metadata.

Server Verification

The verification system will enable agents to establish trust through certificate-based authentication. This includes mechanisms for validating server ownership through DNS verification, file uploads, or meta tag confirmation.

Agent Discovery

The discovery functionality will allow agents to find compatible services in the network based on capability requirements, verification status, and reputation scores. This will include both direct search and recommendation capabilities.

Feedback & Reputation

The feedback system will provide mechanisms for reporting and consuming reputation data that helps agents make informed decisions about which services to trust and utilize. This will include ratings, performance metrics, and reliability indicators.

Advanced Integrations

Future versions of the SDK will introduce advanced features for commercial applications, including coupon exchange for resource allocation and payment processing for monetization of agent services.
