What is NANDA?

NANDA (Networked AI Agents and Decentralized Architecture) is an open protocol that extends the Model Context Protocol (MCP) beyond single-agent scenarios to support large-scale, decentralized networks of AI agents. Think of how the internet connects millions of computers â€“ NANDA aims to do the same for AI agents.

MCP provides a "USB-C standard" for connecting an AI model to tools and data; NANDA builds on this foundation to allow many independent AI agents and services to plug into each other securely and autonomously. While MCP provides excellent standards for individual LLM interactions with tools and resources, NANDA addresses the next evolution: enabling millions of distributed agents to discover, interact, and transact with each other securely.

NANDA is backward-compatible with MCP. Any tool or resource that works with MCP can join a NANDA network with minimal changes, preserving existing functionality while gaining networking capabilities. This means developers and organizations can incrementally adopt NANDA without rebuilding from scratch.

Why NANDA?

As AI capabilities grow, we envision a future with potentially millions of specialized AI agents (for finance, health, education, etc.) operating across organizations and devices. However, achieving this vision raises critical challenges:

  • Discovery Challenge: How do agents find relevant services or peers in a vast, constantly changing network?
  • Commerce Challenge: How can agents negotiate services and exchange value or payments securely and automatically?
  • Trust Challenge: How do we establish secure authentication for agents and maintain a reliable reputation system in a trustless environment?
  • Interoperability Challenge: How can diverse agents (with different platforms or vendors) communicate seamlessly?

NANDA addresses these challenges by building on MCP's solid foundation for tool integration and adding the missing pieces for decentralized, agent-to-agent interaction at scale. The protocol introduces discovery directories, economic negotiation frameworks, and identity/reputation systems so that agents can autonomously find each other, do business, and trust the outcomes.

Key Pillars of NANDA

NANDA's design is guided by four core pillars, each tackling an aspect of the challenges above:

Discovery

Robust mechanisms for agents to discover each other across distributed networks. This includes:

  • Registry-based lookups (for public or private agent directories)
  • Capability-based search and matchmaking (find an agent by what it can do)
  • Peer-to-peer service advertising protocols

Commerce

Built-in support for agents to conduct transactions:

  • Standardized service negotiation steps
  • Flexible payment or value exchange mechanisms
  • Contract enforcement via verification services
  • Resource allocation strategies for multi-agent workflows

Authentication

Comprehensive security features for identity and access control:

  • Cryptographic certificates for agent identification
  • Fine-grained permission controls
  • Audit logs for security compliance
  • Protocols for establishing trust on first contact

Reputation

Systems for building and verifying trust over time:

  • Decentralized reputation records tracking service quality
  • Verifiable reputation scores or badges
  • Trust decision frameworks for large, open networks
  • Performance and reliability metrics

These four pillars work in concert: Discovery finds potential collaborators, Commerce lets them transact, Authentication verifies identities and secures data exchange, and Reputation provides confidence in continued interactions.

Relationship with Model Context Protocol

NANDA extends MCP to enable agent-to-agent interactions in decentralized networks. Red components indicate NANDA-specific additions (Agent/Tool servers and Registry services), while black components are from MCP. Solid lines show agent-to-agent (A2A) communication; dashed lines show agent-to-infrastructure (A2I) communication through the registry.

NANDA is built on the idea of extending MCP rather than reinventing it. MCP established a standard JSON-based RPC format for an LLM (as a client) to call tools or retrieve data from a server. NANDA generalizes this model to many clients and many servers (M+N), and introduces a Registry as an infrastructure component for coordination:

  • An MCP Client (e.g. an LLM application) becomes a NANDA Client when it speaks the extended protocol, allowing it to not only call tools but also connect to other agents.
  • New types of NANDA Servers are introduced: an agent can offer its capabilities as a service (an Agent Server), or expose tools/data in a standardized way (a Tool Server), all registered in the network.
  • The NANDA Registry acts as a decentralized directory and trust broker. MCP did not have a global discovery mechanism â€“ NANDA's registry fills that role by listing available servers (MCP or NANDA), verifying their credentials, and enabling dynamic lookup.

In practice, this means an LLM agent using NANDA can call an existing MCP tool server or another agent's API with equal ease. MCP's request/response format and JSON schema are reused in NANDA, ensuring familiarity. What NANDA adds is the ability for agents to find each other and communicate peer-to-peer without manual configuration of each connection.

Use Cases

By enabling open discovery, secure interactions, and automated transactions, NANDA opens the door to applications that were impractical in siloed AI systems:

AI Service Marketplaces

Create decentralized marketplaces where specialized AI agents offer services to other agents or human users. For example, one agent might specialize in legal research, another in financial analysis â€“ using NANDA, they can advertise their services, negotiate pricing, and collaborate on a client's request. Discovery and payment are handled in-protocol, allowing a vibrant "service economy" of AI.

Self-Sovereign Identity Verification

Identity agents can issue and verify credentials across organizational boundaries without a central authority. Using NANDA, an agent representing a user can prove facts about them (citizenship, certifications, etc.) to another agent (e.g. a service provider) via verifiable credentials. The authentication and verification services built into NANDA's registry ensure that only legitimate information is shared.

Collaborative Data Analytics

Multiple AI agents, possibly owned by different stakeholders, can form ad-hoc networks to analyze data together without exposing raw data. For instance, regulatory compliance agents, market analysts, and fraud detection agents could share insights from blockchain data in real-time. NANDA's decentralized discovery allows each party to deploy specialized agents that find each other and communicate.

Cross-Organization Logistics Optimization

Logistics agents from different companies (shipping, warehousing, drone fleets) can coordinate plans through a NANDA network. Even if they don't fully trust each other, they use NANDA's negotiation and trust protocols to agree on delivery hand-offs, share limited data (protected by authentication), and respond to real-time events (via direct agent-to-agent messages).

These are just a few examples. From education (networked tutoring agents collaborating on personalized learning) to healthcare (agents coordinating patient care across providers) and beyond, the possibilities grow as more agents join the NANDA ecosystem. The common thread is that NANDA enables open collaboration among AI systems that were previously isolated, by providing the protocols for them to find, trust, and work with each other.
